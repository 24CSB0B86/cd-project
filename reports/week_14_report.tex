\section{Integration Architecture}

\subsection{2.1 End-to-End Pipeline Contract}

The formal contract for a complete E2E pass is:

\begin{center}
\texttt{source code} $\xrightarrow{\text{Lexer}}$ \texttt{tokens}
$\xrightarrow{\text{Parser}}$ \texttt{AST}
$\xrightarrow{\text{Semantic}}$ \texttt{diagnostics}
$\xrightarrow{\text{SAST}}$ \texttt{audit report}
$\xrightarrow{\text{Context}}$ \texttt{LLM prompt}
$\xrightarrow{\text{Agent}}$ \texttt{explanation + fix}
$\xrightarrow{\text{Validator}}$ \texttt{fixed code re-parsed: PASS}
\end{center}

The final stage --- re-parsing the AI's suggested fixed code through the Mini-Python parser --- is the key regression contract added in Week 14. If the fix introduces new syntax errors, the test fails.

\section{Bug Inventory \& Resolutions}

Twelve bugs were identified and resolved during integration testing:

\begin{table}[h]
\centering
\begin{tabularx}{\textwidth}{|c|l|l|X|}
\hline
\rowcolor{orange!10}
\textbf{\#} & \textbf{Component} & \textbf{Severity} & \textbf{Description \& Fix} \\
\hline
1 & Lexer & HIGH & DEDENT not emitted at EOF for single-line functions. Fixed by flushing indent stack on \texttt{EOF} token emission. \\
\hline
2 & Parser & HIGH & Chained comparison (\texttt{a < b < c}) caused infinite recursion. Fixed by limiting comparison depth to one level. \\
\hline
3 & Semantic & MEDIUM & \texttt{for} loop variable was not added to symbol table in loop body scope. Fixed by pre-inserting loop var before visiting body. \\
\hline
4 & Semantic & MEDIUM & Function parameters not scoped to function body when the function had no \texttt{return} statement. Fixed in \texttt{\_visit\_FunctionDef}. \\
\hline
5 & Context Provider & LOW & \texttt{serialize\_ast()} omitted \texttt{NoneType} literal nodes, causing incomplete context. Fixed by adding \texttt{NoneConst} visitor. \\
\hline
6 & SAST & LOW & False positive: \texttt{open()} flagged inside a \texttt{\# comment}. Already partially fixed in Week 11; residual edge case with trailing spaces resolved. \\
\hline
7 & CompilerToolbox & MEDIUM & \texttt{reparse\_code()} did not reset the semantic analyzer before re-run, causing stale symbol table entries. Fixed by reinstantiating \texttt{SemanticAnalyzer}. \\
\hline
8 & AgenticGeminiClient & MEDIUM & ReAct loop exited with empty response when \texttt{MAX\_ITERATIONS} was hit without a final natural-language response. Fixed by injecting a fallback summary message. \\
\hline
9 & ChatSession & LOW & \texttt{refresh\_context()} did not update the \texttt{ReasoningLog} reference, causing the XAI tab to display stale trace data after code refresh. Fixed. \\
\hline
10 & ASTVisualizer & LOW & Unicode variable names (e.g., Greek letters) caused Graphviz label encoding errors. Fixed by applying \texttt{repr()} to all node labels. \\
\hline
11 & Streamlit UI & LOW & Security Audit tab's metric cards did not reset between sessions, showing stale counts. Fixed by resetting \texttt{st.session\_state} on ``Clear'' action. \\
\hline
12 & BenchmarkSuite & LOW & \texttt{tracemalloc} left running across tests, causing memory leak in the test runner. Fixed by adding \texttt{tearDown} \texttt{tracemalloc.stop()} call. \\
\hline
\end{tabularx}
\caption{Bug inventory: 12 issues resolved in Week 14}
\end{table}

\section{CI/CD Pipeline}

\subsection{4.1 GitHub Actions Workflow}

A \texttt{.github/workflows/ci.yml} workflow was created with the following stages:

\begin{lstlisting}[language=yaml, caption=GitHub Actions CI pipeline]
name: Project 83 CI

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        python-version: ["3.11", "3.12"]
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: ${{ matrix.python-version }}
      - name: Install dependencies
        run: pip install -r requirements.txt
      - name: Run full test suite
        run: python -m pytest tests/ -v --tb=short
        env:
          GEMINI_API_KEY: ""      # Offline mode - no key needed
      - name: Generate coverage report
        run: python -m pytest tests/ --cov=src --cov-report=xml
      - name: Upload coverage
        uses: codecov/codecov-action@v4
\end{lstlisting}

\section{Regression Testing}

Twelve regression tests were written --- one per bug fix --- using the minimal reproducible input that originally triggered each bug,  ensuring no future commit re-introduces any of the twelve issues.

\begin{lstlisting}[language=Python, caption=Regression test example (Bug \#1)]
def test_lexer_dedent_at_eof():
    """Regression: DEDENT must be emitted at EOF for single-line functions."""
    lexer = Lexer("def f():\n    return 1")
    tokens = lexer.tokenize()
    token_types = [t.type for t in tokens]
    assert TokenType.DEDENT in token_types, "DEDENT missing at EOF"
\end{lstlisting}
